    *   The input grids contain various rectangular blocks of pixels against a background color.
    *   Some blocks are solid (filled with a single color), while others contain patterns or mixtures of colors.
    *   The solid blocks in the examples are consistently surrounded by a one-pixel border of the background color.
    *   The output grids seem to be constructed by selecting only the solid blocks (along with their background border) from the input.
    *   The relative positions of these selected blocks are preserved in the output.

*   **Facts**:
    
```yaml
    task_elements:
      - item: grid
        properties:
          - contains_pixels
          - has_height_and_width
          - background_color (most frequent color or color at corners/edges)
      - item: object
        definition: rectangular block of pixels
        properties:
          - position (top-left corner, bottom-right corner)
          - size (height, width)
          - color (if solid)
          - pattern (if not solid)
          - border (surrounding pixels)
        types:
          - type: solid_block
            criteria: all pixels within the block have the same non-background color
          - type: patterned_block
            criteria: pixels within the block have different colors or include the background color
          - type: background_bordered_block
            criteria: a solid_block where all immediately adjacent pixels outside the block (including diagonals) have the background_color
    actions:
      - action: identify_background_color
        inputs: input_grid
        outputs: background_color
      - action: find_objects
        inputs: input_grid
        outputs: list_of_objects (rectangles with properties)
      - action: filter_objects
        criteria: object must be a solid_block AND a background_bordered_block
        inputs: list_of_objects, background_color
        outputs: list_of_selected_blocks (including border coordinates)
      - action: determine_output_bounds
        inputs: list_of_selected_blocks (including border coordinates)
        outputs: min_row, max_row, min_col, max_col
      - action: construct_output_grid
        inputs: output_dimensions (derived from bounds), background_color, list_of_selected_blocks, input_grid
        process: create new grid, copy selected blocks and their borders from input_grid to the new grid based on relative positions
        outputs: output_grid
    relationships:
      - A solid block is selected only if it is completely surrounded by a 1-pixel border of the background color.
      - The output grid contains only the selected solid blocks and their borders.
      - The relative spatial arrangement of the selected blocks is preserved in the output.
    ```


*   **Natural Language Program**:
    1.  Identify the background color of the input grid (this is typically the color of the corners or the most frequent color).
    2.  Scan the input grid to locate all rectangular blocks composed of a single, non-background color (solid blocks).
    3.  For each solid block found, check if it is completely enclosed by a one-pixel thick border consisting solely of the background color. Consider adjacent pixels (horizontally, vertically, and diagonally) just outside the block's perimeter.
    4.  Select only those solid blocks that meet the bordering criteria. Keep track of the coordinates of these blocks *including* their one-pixel background border.
    5.  Determine the minimum bounding box (minimum row, maximum row, minimum column, maximum column) that encompasses all the selected blocks and their borders.
    6.  Calculate the height and width of the output grid based on this bounding box.
    7.  Create a new grid of the calculated dimensions, initially filled with the background color.
    8.  For each selected block (with its border), copy the corresponding region from the input grid into the output grid, adjusting its position relative to the top-left corner (minimum row, minimum column) of the determined bounding box.
