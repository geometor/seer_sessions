*   **Perception:**
    *   The input grids consist of horizontal stripes of solid colors. The width of the input grid seems irrelevant to the transformation.
    *   The output grids are significantly larger square grids composed of nested square frames, culminating in a solid square center.
    *   The colors used in the output are the same as those present in the input stripes.
    *   There's a direct relationship between the sequence of stripes in the input (read top-to-bottom) and the sequence of frames in the output (read outside-to-inside).
    *   The thickness of each horizontal stripe in the input (number of rows) determines the thickness of the corresponding frame in the output, or the size of the central square for the last stripe.
    *   The size of the output grid is determined by the sum of the thicknesses of the input stripes. Specifically, if the thicknesses are t1, t2, ..., tN, the output dimension is `2 * (t1 + ... + tN-1) + tN`.

*   **Facts:**
    
```yaml
    task_type: transformation
    input_features:
      - grid_representation: 2D array of integers (colors)
      - structure: Horizontal stripes (contiguous blocks of identical rows)
      - properties:
          - stripe_color: The color of the pixels in a stripe.
          - stripe_thickness: The number of rows in a stripe.
          - stripe_order: The top-to-bottom sequence of unique stripes.
    output_features:
      - grid_representation: 2D array of integers (colors)
      - structure: Nested square frames with a central square block.
      - properties:
          - frame_color: Color of a frame, corresponding to an input stripe color.
          - frame_thickness: Thickness of a frame, corresponding to the thickness of an input stripe.
          - center_color: Color of the central block, corresponding to the last input stripe color.
          - center_size: Dimensions of the central block (thickness x thickness), corresponding to the thickness of the last input stripe.
          - grid_size: Square dimensions determined by input stripe thicknesses.
    transformation_logic:
      - action: Identify unique horizontal stripes and their properties (color, thickness) from the input.
      - action: Calculate the output grid size based on the sum of stripe thicknesses.
      - action: Construct the output grid by drawing nested frames.
      - relationship: The i-th stripe from the top in the input corresponds to the i-th frame/center block from the outside in the output.
      - relationship: The thickness of the i-th input stripe determines the thickness/size of the i-th output frame/center block.
    ```


*   **Natural Language Program:**
    1.  Analyze the input grid to identify the sequence of horizontal stripes. A stripe is a contiguous block of one or more identical rows.
    2.  For each stripe, record its color and its thickness (the number of rows it occupies). Let this sequence be `(color_1, thickness_1), (color_2, thickness_2), ..., (color_N, thickness_N)`.
    3.  Calculate the dimension `D` for the square output grid: `D = (2 * sum(thickness_i for i from 1 to N-1)) + thickness_N`.
    4.  Create a new square grid of size `D x D`.
    5.  Initialize a variable `current_offset` to 0.
    6.  Iterate through the stripes from `i = 1` to `N-1`:
        a.  Get the color `color_i` and thickness `thickness_i`.
        b.  Draw a square frame within the output grid using `color_i`. This frame has a thickness of `thickness_i`. Its outer boundary starts at row `current_offset` and column `current_offset`, and ends at row `D - 1 - current_offset` and column `D - 1 - current_offset`.
        c.  Update the offset: `current_offset = current_offset + thickness_i`.
    7.  Get the color `color_N` and thickness `thickness_N` of the last stripe (the N-th stripe).
    8.  Fill the central rectangular area of the output grid with `color_N`. This area starts at row `current_offset` and column `current_offset`, and has dimensions `thickness_N x thickness_N`.
    9.  Return the constructed output grid.
